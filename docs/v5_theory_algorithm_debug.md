## 第一部分: 理论创新可行性 (诚实评判)

### 1.1 什么是"真理论创新" (调研结论)

材料ML领域的"真理论创新" (非组合) 分几类:
1. **新物理约束** (PRL 2021 Beucler, ICML 2025 DFST): 把守恒律/对称性硬编码进网络架构
2. **新不确定性理论** (NeurIPS 2024 自校准conformal): 条件有效的预测集
3. **新描述符理论** (SISSO): 从数据发现物理解析公式
4. **新外推理论** (Nature Comm Mater 2025): 迁移学习的外推保证

### 1.2 我们的诚实定位: 理论创新不现实 (本科生单机)

**原因**:
- 真理论创新需要深度物理+数学推导 (如等变网络的表示论), 超出材料本科生能力
- 需要大量计算资源 (PINN/等变GNN需多GPU), 我们只有单RTX 4060
- 顶刊理论创新周期通常1-2年, 我们3个月

**但有两个"准理论"方向可做** (调研发现的轻量级理论贡献):

#### 准理论A: 形成能物理分解 (Kapustinskii/Madelung + ML残差)
**文献支撑**: Madelung能量是离子固体形成能的主导贡献
  ([ScienceDirect](https://www.sciencedirect.com/topics/engineering/madelung-energy),
   [PCCP 2021](https://pubs.rsc.org/en/content/articlehtml/2021/cp/d1cp03280a))
**做什么**: E_formation = E_Madelung(可解析算) + E_共价修正(ML学) + E_畸变(ML学)
  把形成能分解为有物理含义的分量, ML只学"偏离经典离子模型"的部分。
**理论性**: 有物理推导 (Madelung常数 + Born-Landé), 非纯黑盒。
**可行性**: ★★★ (需算Madelung能量, pymatgen可估算)
**创新性**: 中 (有人做过类似分解, 但在钙钛矿+conformal框架内是新的)

#### 准理论B: SR适用判据的形式化
**做什么**: 把我们经验的"SNR≥2 + 线性R²≥0.5"判据做理论推导——
  在什么数据分布下, 符号回归的期望R²有解析下界?
**理论性**: 有概率论推导 (与PAC学习理论相关)。
**可行性**: ★★ (推导难度高, 但可尝试)
**创新性**: 中高 (据调研无人在材料ML做这个)

### 1.3 结论: 理论创新建议

**推荐做准理论A (形成能物理分解)**: 它给项目一个物理理论锚点,
论文可写"我们提出形成能的物理可分解模型", 这是超出纯ML的贡献。
若时间不够, 保持当前"方法学创新"定位 (SR适用判据+conformal+LOEO),
对三区已够。

#### 方向1: GBDT Stacking (LGB+XGB+CatBoost) ★★★★

**预期**: R²持平或略升, 但物理可解释性大幅提升 (Madelung分量有解析公式)
**创新性**: 这是真"算法+物理"创新

#### 方向3: 多任务学习 (形成能+凸包能联合) ★★★
**文献**: 共享表征可互相促进
**预期**: 凸包能R²可能 +0.01-0.02 (借形成能信息)
**可行性**: 中

#### 方向4: 残差Stacking (conformal作为元学习器) ★★
**预期**: 不确定

### 2.3 最值得做的: 方向1 (Stacking) + 方向2 (物理分解)

**Stacking** 是最快提R²的, **物理分解** 是提创新性的。两者可并行。

## 第三部分: 全量Debug结果 (重要!发现真bug)

### ★ 发现并修复的真Bug: d/f电子计算错误

**Bug**: `features.py` 的 `_avg_d_electrons` 用 `sum(orb[2] for orb in es if "d" in orb[1])`,
  但 `full_electronic_structure` 包含**所有内层轨道**, 导致:
  - Ac ([Rn]6d¹) 得 d_electrons = **31** (3d¹⁰+4d¹⁰+5d¹⁰+6d¹ = 31)
  - 正确应为 **1** (只数价层6d)
  - 影响: phys_a/b_site_d_electrons 系统性偏大 (max 32 而非 10)
  - f电子同理 (max 20 而非 14)

**根因**: pymatgen `full_electronic_structure` 含内层轨道, 代码错误地全数了。
**修复**: 改为只数**最高主量子数**的d/f轨道 (价层)。
**验证**: Fe→6, Ti→2, La→1, Gd→(d1,f7), Ce→(d1,f1), Cu→10 全部正确。
**影响**: R² 0.916→0.914 (-0.002, 可忽略, 因d电子非主导特征)。
**意义**: 数据现在**物理正确**。之前d_electrons=32是无意义的, 可能误导SHAP/SR分析。
