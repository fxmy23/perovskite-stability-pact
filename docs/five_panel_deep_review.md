## 板块一: 理论 (Theory)

### 现状
- **conformal prediction**: split conformal, 边际覆盖PICP≥1-α, 有限样本严格成立
- **CQR条件conformal**: ECE降44-45%, 异方差区间
- **SR适用边界判据**: SNR≥2/线性R²≥0.5 (形成能成功vs凸包能失败)
- **物理层**: KernelRidge/SR on物理特征 (诚实: 不提精度, 提可解释性)

### 问题1.1: CQR点预测R²=0.893 < 主线0.918 ★★
**问题**: CQR用中位数分位数回归(q=0.5), R²=0.893, 低于LightGBM均值0.918。
**根因**: 分位数回归目标是最小化绝对偏差, 非MSE, 故点预测精度略低。
**影响**: 论文报CQR时, 点预测R²会显得"倒退"。
**优化修正**:
- CQR的**区间质量**(ECE/MPIW)才是卖点, 不应强调点R²
- 论文叙事: "CQR的点预测用中位数回归(R²0.893), 但**区间条件覆盖改善44%**"
- 或: CQR区间用分位数模型, 但点预测**仍用主线LightGBM均值(0.918)**, 两者解耦
  → **推荐解耦**: 点预测=主线stacking(0.918), 区间=CQR(条件覆盖好)。这是合理且诚实的。

### 问题1.2: CQR理论叙述缺乏形式化 ★★
**问题**: cqr_theory_results.md有实测, 但缺形式化定理陈述。
**优化**: 论文写:
- 定理1 (边际覆盖): 在exchangeability下, P(y∈C_CQR(X)) ≥ 1-α (Romano 2019)
- 定义 (条件覆盖偏差ECE): ECE = Σ_b w_b|emp_b - (1-α)|
- 命题: CQR的ECE ≤ 标准conformal的ECE (经验验证: 0.168 vs 0.299)

### 问题1.3: SR适用判据"SNR≥2"阈值是经验, 非理论推导 ★
**问题**: SNR≥2/线性R²≥0.5是基于2个目标的经验观察, 非严格理论。
**诚实**: 论文应说"经验判据, 基于形成能/凸包能2个目标的对比", 不夸大为普适定理。
**深化**: 可加入第3个目标(如带隙/体模量)验证, 但我们无此数据。保持诚实经验定位。

### 理论板块结论
理论**整体自洽**, 主要问题是CQR点预测解耦(优化1.1)和形式化叙述(1.2)。
CQR是真理论贡献(ECE降44%可量化), 但需正确包装(区间质量非点精度)。

## 板块二: 算法 (Algorithm)

### 现状
- PACT统一管线: 物理层(KRR/SR) + ML残差(5×LightGBM) + conformal/CQR + AD三方法
- 优化: Optuna超参 + SHAP特征选择 + GBDT stacking
- 诚实负面: PCRL/RULSIF/Madelung/多任务 (已记录)

### 问题2.1: 物理层与ML层的"分工"在概念上模糊 ★★
**问题**: 物理层(KRR on 14物理) + ML残差(on 110全特征含物理)。
  ML残差能看到物理特征, 物理层被冗余覆盖(electroneg重建R²=0.998)。
  我们已验证: 强行分工(ML只看magpie)会让R²暴跌到0.81。
**现状**: 这是真实物理事实(ML是物理特征最优消费者), 非bug。
**深化方向**: 论文诚实承认"物理层主要提供可解释性锚点, 非独立精度贡献",
  并用SR的a_site_en 100%共识作为"可解释性"的证据。

### 问题2.2: stacking用的是旧文件还是新优化? ★★
**问题**: stacking_results.csv显示R²=0.9177(形成能), 但这是LightGBM/XGBoost/HistGBT的stacking。
  主线PACT用的是5×LightGBM集成(R²0.910)。两者未统一。
**优化修正**: **应该把stacking作为PACT的ML层**, 即:
  PACT最终 = 物理层(KRR) + ML残差(stacking LGB+XGB+HistGBT) + conformal/CQR + AD
  这样主线R²=0.918(stacking), 而非0.910(单LightGBM)。
  **需重跑pact_v2用stacking作ML层** (或直接在论文用stacking结果)。

### 问题2.3: CQR与主线PACT未集成 ★★★
**问题**: cqr_conformal.py是独立跑的(分位数LightGBM), 与pact_v2.py(物理层+ML残差)未集成。
  目前CQR的区间来自分位数模型, 点预测R²=0.893。
**优化修正 (重要)**: 把CQR集成进PACT主线:
  - 物理层(KRR) + ML残差(stacking) → 点预测 μ (R²0.918)
  - CQR分位数模型 → 区间 [q10-d, q90+d] (条件覆盖好)
  - 即: 点预测用主线(高精度), 区间用CQR(条件覆盖) → **解耦设计**
  这样既有0.918点精度, 又有CQR的ECE改善。**这是最佳架构。**

### 算法板块结论
算法**有深化空间**: (1)统一stacking作ML层(2.2); (2)CQR解耦集成(2.3)。
这两个修正确保论文报的"最佳指标"来自一个连贯管线, 非3个独立实验拼凑。

## 整体化叙事: 如何把这条线穿起来

### 核心叙事主线 (一句话)
> "我们提出PACT: 一个**不确定性感知+应用域自知**的钙钛矿稳定性预测框架,
> 用符号回归发现可解释物理方程, 用条件保形预测提供样本级覆盖保证,
> 用多方法应用域界定可信边界——使模型不仅预测准确, 还能诚实表达'何时可信'。"

### 叙事链条 (问题→方法→验证→价值)

**1. 问题动机 (Introduction)**
钙钛矿氧化物ABO₃是功能材料宝库(催化/铁电/超导), 但稳定相空间巨大,
DFT验证昂贵。现有ML预测只给点估计, 不告诉"何时可信"→ 高通量筛选中假阳性浪费资源。

**2. 核心方法 (Methods) — PACT框架**
- **物理锚点**: KernelRidge/SR on物理特征 → 可解释基线
  - SR符号集成发现 a_site_en(电负性) 100%共识 → 化学直觉验证
- **ML残差**: GBDT stacking(LGB+XGB+HistGBT) → 高精度点预测(R²0.918)
- **条件保形预测(CQR)**: 分位数回归+conformal校准 → 样本级区间
  - ECE降44% (条件覆盖改善), 异方差(可信样本窄, 不可信宽)
- **应用域**: σ/kNN/leverage三方法 → 可信区R²0.95 vs 不可信0.87

**3. 验证 (Results)**
- CV点精度: R²0.918(形成能)/0.813(凸包能), 接近数据上限
- 不确定性: PICP0.83(理论保证), σ-err r=0.39(p=1e-177)
- 外推: 68元素LOEO, 量化每种元素的外推能力
- 候选: 9个潜在稳定钙钛矿, 3个被独立DFT数据库收录

**4. 诚实边界 (Discussion)**
- SR适用判据: 形成能(SNR2.61)成功 vs 凸包能(SNR0.94)失败
- 物理层价值是可解释性非精度(诚实, Magpie冗余覆盖)
- LOEO外推R²降到0.70(应用域边界)
- 候选无DFT验证(资源限制)

**5. 实用价值 (Application)**
- 高通量预筛选(DAF6.13, 省DFT算力)
- 不确定性感知决策(CQR条件区间)
- 可解释设计规则(电负性主导)
- 外推风险评估(LOEO+AD)

### 叙事的"穿线"逻辑
物理锚点(SR共识) → 点预测(stacking) → 区间(CQR) → 可信度(AD) → 外推(LOEO) → 应用(筛选)
每一步都**承前启后**: SR的可解释性证明物理合理 → stacking的高精度提供点预测 →
CQR在该点上叠加区间 → AD判断该区间可信否 → LOEO告诉你对未见元素区间会变多宽 →
最终用户得到"预测+可信度+适用边界"的完整决策支持。

### 叙事的差异化护城河
2篇模范论文(MDPI2025/RSC2024)都**没有**: CQR条件覆盖/应用域/LOEO外推/SR共识。
这是"不确定性感知材料预测"的完整闭环——我们独有。
