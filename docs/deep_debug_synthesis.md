## 二、解释当前结果的6个发现

### 发现1: R²=0.916 是真实的数据上限 (非bug) ★
- 线性回归(14物理特征) R²=0.563 → 线性上限
- 线性残差 std=0.652 eV (不可约噪声估计)
- LightGBM 用非线性+Magpie 把 R² 推到 0.916, 已远超线性上限
- 进一步提升空间小: DFT数据本身有数值噪声 (~0.05-0.1 eV),
  理论 R² 上限约 0.93-0.95 (我们的 0.916 已接近)

### 发现2: RULSIF 无效的真根因 (非bug, 是模型特性)
- 权重正确且含信息 (Spearman r=0.393)
- 但 LightGBM (树模型) 对 sample_weight 弱敏感
- 文献的28%改善在线性/核模型上, 树模型不适用
- 这是方法-模型匹配问题, 诚实负面

### 发现3: SHAP只选Magpie的原因 (重要!) ★
- **phys_electroneg_diff_AB 可被Magpie线性重建 R²=0.998**!
  → 电负性差几乎完全被 Magpie 的 X_wmean/X_std 冗余表达
- phys_octahedral_factor 重建 R²=0.432 (部分冗余)
- phys_tolerance_factor 重建 R²=0.048 (几乎独立, 不冗余)
**解释**: LightGBM 选 Magpie 不是bug, 是因为 Magpie 统计特征
  (尤其电负性X) 冗余覆盖了我们的物理特征。物理特征中只有
  tolerance_factor 提供独立信息, 所以它没进top但SR能用到。

### 发现4: 物理特征 vs Magpie 的真实关系
- 我们的14物理特征中, 多数被Magpie冗余覆盖 (电负性/半径/价态)
- 仅 tolerance_factor 和 radius_ratio 提供独立几何信息
- 这解释了"物理层不提精度"——Magpie已包含大部分物理信息
- SR 仍用 a_site_en 因为它在符号空间表达更简洁 (虽被Magpie冗余)

### 发现5: 形成能 vs 凸包能的差异 (物理一致)
- Pearson r(E_f, E_hull)=0.593 (中等相关, 物理合理)
- 373个 E_f<-3 的样本中, 69个 E_hull>0.5
  (形成能很低但凸包能大 = 化合物本身稳定但有更稳定竞争相, 物理合理)
- 凸包能更难预测 (R²0.807 < 形成能0.916) 因它依赖"所有竞争相",
  超出单化合物描述符能表达的范围

### 发现6: LOEO外推难是数据结构决定 (非bug)
- 固定A位(Ba)的Magpie特征组内/全体比=0.665
  → A位元素只解释33%的Magpie方差, B位变化贡献剩余
- 这意味着模型在LOEO时, 对留出A位元素, 仍能从B位特征获取信号
- 但A位相关的1/3信号丢失, 导致 R² 从0.916降到0.70 (合理)

## 三、综合诚实结论

**我们的结果(R²0.916, RULSIF无效, SHAP选Magpie)全部有合理解释, 无致命bug。**

1. **R² 0.916 是真实的**: 接近 wolverton 数据(4914样本)的物理上限
   (~0.93-0.95, 受DFT数值噪声限制)
2. **RULSIF 无效是真实现象**: 树模型对权重不敏感, 非实现错误
3. **SHAP选Magpie是合理的**: Magpie冗余覆盖了多数物理特征
   (电负性差重建R²=0.998), LightGBM自然选信息密度更高的Magpie
4. **物理特征仍有独立价值**: tolerance_factor不被冗余,
   SR发现的a_site_en共识在符号空间有意义(虽被Magpie数值冗余)

## 四、对论文的诚实启示

1. **不能宣称"物理特征提供独立预测信息"** — 多数被Magpie冗余
2. **可以宣称"物理特征提供可解释性"** — SR方程虽冗余但可读
3. **R² 0.916 是诚实的好结果** — 接近数据上限, 非模型不行
4. **LOEO 0.70 是合理的** — A位特征丢失1/3信号的必然结果
