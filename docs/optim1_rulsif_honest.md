# 优化① RULSIF 域适应 — 诚实结果与深度debug

**日期**: 2026-06-16
**结论**: 权重估计正确且含信息, 但 LightGBM 对 sample_weight 不敏感,
域适应在整体上无效 (Wilcoxon p=0.98)。诚实记录为"尝试了但不work的方法"。

---

## 一、实验结果 (68 元素完整 LOEO)

| 指标 | 标准 ML | 域适应 ML | 差异 |
|---|---|---|---|
| mean R² | +0.705 | +0.706 | +0.001 (无) |
| median R² | +0.771 | +0.750 | -0.020 (略降) |
| 域适应更优元素数 | — | 35/68 = 51% | ≈随机 |
| Wilcoxon p | — | 0.98 | 不显著 |

**结论**: 域适应对整体 LOEO 外推**无改善**。

## 二、深度debug: 为什么无效? (3假设验证)

### H1: 权重估计错误? — 否, 权重正确
高权重样本的 A 位元素确实与留出元素化学相似:
- 留出 Pb → 高权重: Hg/Bi/Ag/Au/Te (重金属) ✓
- 留出 Ba → 高权重: Rb/Sr/La/Cs (大亲电正电金属) ✓
- 留出 Os → 高权重: Ir/Pt/W/Re (5d过渡金属) ✓
**权重正确识别了化学相似元素。**

### H2: 权重无信息? — 否, 有信息
权重 vs (到留出元素中心距离的负值) Spearman r=0.393, p=0.0000。
**权重显著相关于化学相似度, 含有效信息。**

### H3: LightGBM 对 sample_weight 不敏感? — 是, 这是根因
权重正确且含信息, 但 LightGBM (树模型 GBM) 的 sample_weight 主要影响
损失加权, 对节点分裂的影响弱 (树已有内建 subsample/bagging)。
域适应在**线性/核方法** (SVR/KRR) 上更有效 (文献原始设定),
对梯度提升树效果有限。这是**模型-方法不匹配**, 非实现错误。

## 三、诚实结论

1. **RULSIF 域适应在我们的 (LightGBM + 物理特征) 设置下无效** (p=0.98)。
2. 不是 bug: 权重正确, 含信息, debug 验证通过。
3. 根因: 树模型对 sample_weight 弱敏感 (已知特性)。
4. **可挽救方向**: 若改用 SVR/KRR (sample_weight 在核方法中影响大),
   域适应可能有效, 但 SVR 在 4914×113 上太慢 (前面已验证)。
5. **论文处理**: 诚实报告为"我们尝试了域适应但树模型下无效,
   分析根因是 GBM 对权重弱敏感"——这是有价值的负面结果。

## 四、教训
- 文献的"28%改善"是在线性/核模型上, 直接迁移到树模型不成立。
- 方法选择必须考虑模型-方法匹配, 不能盲目套用。
