## 一、审查发现的严重问题 (必须正视)

### 🔴 问题 A: DAF=6.37 "超 M3GNet" 是田忌赛马 (致命伤)

**审查结论**: DAF=6.37 数字本身正确(计算无误、OOF 严谨、无泄露),
但 **"超过 M3GNet 2.70" 这个 claim 不能成立**:

1. **数据集不可比**: wolverton 是钙钛矿单相家族的网格遍历(化学空间极度平滑),
   M3GNet 的 2.70 在跨结构类型/跨化学空间的异构数据集上——**难易程度差 3 倍以上**
2. **稳定率差异**: wolverton 稳定率 9.4%, Matbench Discovery 约 30%+。
   DAF 对基础率极度敏感——基础率越低 DAF 越容易刷高
3. **Recall=42.9% 是硬伤**: 模型漏掉了 57% 的真稳定材料,
   作为"完整发现工具"不合格

**审稿人若懂行, 这个 claim 是论文的致命伤。**

### 🔴 问题 B: 架构有两个 P0 级自相矛盾

**矛盾 B1: PGML 用 Ridge, PCRL 用 KernelRidge, 互相打架**
- pgml.py 的注释自己论证了"KernelRidge 过拟合→残差坍缩→R²变负"
- 但 pcrl.py 偏偏用回了 KernelRidge
- 同一篇论文两个物理基线, 数字不可直接比较

**矛盾 B2: features.py 落盘时已全量填充, 使 utils.py 的防泄露失效**
- 我们以为修了 P0-1 泄露(改 utils.py)
- 但 features.py 的 build_features 在落盘前已经做了全量 median 填充
- **泄露修复未真正闭环**

### 🟡 问题 C: 其他自洽性隐患
- PCRLModel 类是死代码(run_pcrl_comparison 重写了逻辑)
- PHYS_FEATURES 列表有 3-4 份硬编码副本
- Magpie 维度文档写 132 实际 96
- 4 个模块绕过 get_feature_cols

## 二、回答你的四个问题

### Q1: 结果是否可信? 能否解释来龙去脉?

**诚实回答: 大部分可信, 但有两处"数字漂亮但故事存疑"**

| 结果 | 可信度 | 说明 |
|------|--------|------|
| 形成能 R²=0.91 | ✅ 可信 | 多次验证稳定, 无泄露 |
| 凸包能 R²=0.80 | ✅ 可信 | 同上 |
| LOEO 外推 | ✅ 可信 | 实现正确 |
| SHAP 物理解释 | ✅ 可信 | TreeExplainer 标准 |
| PGML 物理贡献 | ⚠️ 待修 | 公式已修但基线矛盾未解 |
| **DAF=6.37** | ⚠️ **数字对, 故事错** | 不可与 M3GNet 直接比 |
| 新候选 9 个 | ✅ 可信 | 特征对齐已修 |

**结论**: 性能数字可信, 但 DAF 的对比叙事必须改。Recall=42.9% 必须诚实报告。

### Q2: 论文深度够吗? 要不要加 GNN?

**诚实回答: 当前深度处于三区下限, 不加 GNN 难以"出彩"**

当前论文的"深度"分析:
- 方法层: PCRL 是创新, 但 feature_penalty 是启发式(SHAP 验证虽有效但变化幅度仅 2.6%)
- 实验层: 有 LOEO + 分类指标 + SHAP, 比较完整
- **缺什么**: 缺一个"硬核"的方法对比——如果只和 LightGBM 比, 审稿人觉得"没有深度学习方法, 不够 modern"

**GNN 的价值(你的 4060 能跑 CGCNN)**:
1. CGCNN 用晶体结构是**合法的**(GNN 从结构学习表示, 不是用 DFT 标量特征)
2. 补上"组成特征 R²=0.23"的短板——GNN 在 matbench 上能到 0.79
3. **PCRL(组成) vs CGCNN(结构)的对比**是一个有深度的方法学讨论
4. 体现"我们既懂传统ML又懂深度学习"

**我的判断: 加 GNN, 但定位为"方法对比"而非"主方法"**。
主方法仍是 PCRL(无GPU依赖、更通用), GNN 作为 upper bound 对比。
这样论文有"浅(组成)+ 深(结构)"两层, 深度足够。

### Q3: 项目完整性/自洽性?

**诚实回答: 存在 2 个 P0 矛盾, 必须修复后才能说"完整"**

详见问题 A/B。修复方案见第四节。

### Q4: 逻辑是否层层递进?

**当前逻辑链** (有断裂):
```
数据(wolverton) → 特征(magpie+phys) → 基准(LightGBM)
  → PGML(物理+ML) → PCRL(约束残差)  ← 这里基线矛盾!
  → 分类指标(DAF)  ← 这里对比不公平!
  → 材料发现(候选)
```

**修复后的理想逻辑链** (层层递进):
```
数据 → 组成特征 → 基准 → PCRL(统一KernelRidge基线+SHAP约束)
  → 结构特征CGCNN(深度对比) → 分类指标(诚实, 不与M3GNet直接比)
  → 外推LOEO → 适用域 → 材料发现
```

## 三、GNN 决策: 加, 怎么加最出彩

### 为什么加 CGCNN
1. **4060 8GB 完全可行** (有 2024 论文用同款卡)
2. 补深度短板(从"只有传统ML"到"传统ML+深度学习对比")
3. 结构特征在 GNN 里是合法输入(非泄露)
4. PCRL(组成) vs CGCNN(结构) 是有深度的方法学叙事

### 怎么加最出彩 (不是简单"也跑个GNN")
**创新点: PCRL + CGCNN 的融合**
- CGCNN 学结构表示, PCRL 学组成+物理
- 两者预测加权融合, 或 CGCNN 特征喂给 PCRL 残差
- 这才是"深度学习+物理引导"的真正结合

### 风险与对策
- 风险: PyTorch 环境搭建(可能遇 Windows 兼容问题)
- 对策: 先装 CPU 版验证流程, 再上 GPU
- 时间: CGCNN 训练 1-3 小时, 环境搭建 0.5-1 天

## 四、修复 + 加深 的完整路线图

### 第一阶段: 修复自洽性 (必须先做, 半天)

7. 用 matbench_perovskites 的结构数据训练 CGCNN

### 第三阶段: 论文打磨 (1天)
