## 一、审查发现的剩余问题 (按严重度)

### 🔴 必须修复的 P0 (不修则投稿必被质疑)

#### P0-1: CGCNN 图构造 bug + 评估乐观偏差
- **图构造**: `nbr.index` 在 pymatgen 新版中语义不确定, 可能连错节点
- **多 seed 假象**: 3 个 seed 只改了模型初始化, **数据划分完全相同**, std 严重低估
- **划分不当**: 普通随机划分在结构相似的钙钛矿上造成乐观偏差
- **early stopping**: 监控训练 loss 而非验证 loss, 等于选过拟合最严重的点
- **edge_attr 白算**: 算了边特征但 GCNConv 没用到

**影响**: CGCNN R²=0.80 被高估, 多 seed std 不可信

#### P0-2: Stacking 单层 CV 评估泄露
- 用同一组 KFold 既生成 OOF 又评估元模型
- 标准做法要求**嵌套 CV** (外层评估 / 内层生成 OOF)
- 后果: stacking "超越单模型"的结论是评估伪影

#### P0-3: PCRL v2 超参选择泄露
- SHAP-guided penalty 选择用了**全量数据**
- 然后 CV 评估——等于"用测试集选超参再在同一测试集评估"
- 后果: pcrl_v2 的 R² 带乐观偏差, 对比表内不公平

#### P0-4: PHYS_FEATURES 未更新 (架构不自洽)
- 新增 4 个电子构型特征是物理特征, 但没进 PHYS_FEATURES 列表
- 导致 PCRL/PGML 的"物理贡献占比"口径错误
- 叙事矛盾: 一边说"d/f 电子是物理本质", 一边把它们算作 ML 侧

### 🟡 应该修复的 P1

| 问题 | 影响 |
|------|------|
| 文档残留旧数字(DAF 6.47 vs 6.44, 物理60% vs 78%) | 审稿人对照发现不一致 |
| 多 seed 同时改划分+模型种子 | std 含两种噪声, 语义混淆 |
| CGCNN vs PCRL 跨数据集对比"自说自话" | self-serving 嫌疑 |

## 三、SCI 三区对标评判

### Computational Materials Science 三区审稿人会怎么看?

| 审稿维度 | 我们的状态 | 三区门槛 | 达标? |
|----------|-----------|----------|-------|
| **数据严谨** | 泄露闭环已修, 但 stacking/PCRL v2/CGCNN 仍有评估泄露 | 必须无泄露 | ⚠️ 边缘 |
| **方法创新** | PCRL(组成+SHAP约束) + CGCNN(结构) 两层 | 需明确创新 | ✅ 达标 |
| **评估严谨** | 多 seed±std, 但 CGCNN std 不可信 | 需多 seed | ⚠️ 部分 |
| **基线对比** | matbench 同数据集 LightGBM 0.228 vs CGCNN 0.80 | 需公平对比 | ✅ 达标(同数据集) |
| **物理洞见** | SHAP + 电子构型 + LOEO 适用域 | 加分项 | ✅ 达标 |
| **可复现** | 代码+数据完整, 无单元测试 | 需代码开源 | ✅ 达标 |

### 诚实评判

**当前状态: 距离三区还有一步之遥。**

核心实验(基准+PCRL+分类+LOEO+SHAP)是扎实的, 但新增的三个"加分项"(stacking/PCRL v2/CGCNN)**都有评估方法学问题**, 如果带着这些问题投稿, 审稿人一旦深入检查就会质疑。

**关键洞察**: 我们不需要"更多花哨的方法", 而需要**把现有方法做对**。
- Stacking 改嵌套 CV 即可
- PCRL v2 改嵌套 CV 选 penalty 即可
- CGCNN 修图构造 + 改结构划分 + 修 early stopping 即可
- PHYS_FEATURES 加 4 个特征重跑即可

这些修复都是**确定性的**, 不需要新想法, 只需要严谨。

## 四、解决方案 (切实可行)

### 方案 A: 修复全部 P0 (推荐, 2-3 天)

1. **PHYS_FEATURES 更新** (10分钟): 加入 4 个电子构型特征 → 重跑 PCRL/PGML
2. **Stacking 嵌套 CV** (半天): 外层评估 / 内层生成 OOF
3. **PCRL v2 嵌套 CV 选 penalty** (半天): penalty 选择在内层训练折做
4. **CGCNN 修复** (1天):
   - 用 `get_all_neighbors` 替代逐 site 调用
   - early stopping 用验证集 loss
   - 多 seed 改变数据划分(不只改模型种子)
   - 加结构去重/scaffold 划分(可选)

### 方案 B: 降级处理 (保守, 半天)

如果时间紧, 可以:
- **去掉 stacking 和 PCRL v2** (有评估泄露的两个)
- 只保留 PCRL v1(无泄露) + CGCNN(修图构造即可)
- 论文聚焦"组成特征 PCRL + 结构特征 CGCNN"两条主线
- 物理贡献占比用修复后的 PHYS_FEATURES

### 我的推荐: 方案 A
理由: 这些修复不是"锦上添花", 而是"雪中送炭"。
带着评估泄露投稿 = 浪费一次评审机会。
修复后的 stacking/CGCNN 才是真正可信的加分项。
