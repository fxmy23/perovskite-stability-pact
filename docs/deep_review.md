## Q2: 是否应该自创并优化算法, 用专门章节描述?

### 调研结论: **强烈建议, 这是三区→二区的关键**

文献证据:
- npj Computational Materials 2025 接受的"physics-informed Bayesian optimization"
  ([来源](https://www.nature.com/articles/s41524-025-01522-8))正是因为有**原创物理引导算法**
- 纯应用型论文(换数据集重跑模型)在三区处于**边缘**, 审稿人常以"创新性不足"拒稿

### 具体建议: 把 PGML 写成"原创方法"章节

当前 PGML 实现只是"Ridge基线 + LightGBM残差"——这太简单, 难称原创。
应该升级为**有数学推导的物理引导方法**, 例如:

**建议方案: Physics-Constrained Residual Learning (PCRL)**
1. **数学公式**: 推导 PGML 的目标函数
   $\hat{y} = f_{phys}(\mathbf{x}_{phys}) + g_{ML}(\mathbf{x}_{all})$
   并给出残差正则化项 $\mathcal{L} = \|y - \hat{y}\|^2 + \lambda\|g_{ML}\|^2$
2. **物理约束**: 容忍因子的解析形式嵌入(不只是输入特征)
3. **与现有方法对比**: 对比"纯ML"、"纯物理"、"标准残差"、"我们的PCRL"
4. **理论分析**: 证明物理基线降低了残差复杂度→减少过拟合

这样写一个 2-3 页的方法章节, 论文从"应用型"升级为"方法贡献型",
是三区审稿人最看重的。**这是大幅进步的最大空间所在。**

## Q3: 效果不好就不达标吗?

### 回答: **部分对, 但不完全对。**

**对的部分**: 纯粹的低性能(R²=0.3)确实难发表, 审稿人会质疑"模型没用"。

**不对的部分**: 三区接受"中等性能+强洞察"的论文。先例:
- Open Catalyst Experiments 2024 ([OCx24](https://arxiv.org/html/2411.11783v1))
  明确报告"modest performance"但仍发表, 因为有价值的是**洞察和发现**
- npj 标准第(2)(3)条强调"strong evidence"和"important to field",
  不是"highest accuracy"

**对本项目的具体判断**:
- 形成能 R²=0.91 → 性能足够 ✓
- 凸包能 R²=0.80 → 性能中等, 但配合LOEO/SHAP/材料发现, 可接受 ✓
- matbench R²=0.23 → 太低, **不能放进论文主表**, 只能在"局限性"讨论
- 跨数据集 R²=-10.75 → **必须作为负面发现诚实报告**, 体现严谨

**所以**: 当前结果**够发表**, 但需要精心组织叙事——
把强项(形成能+PGML+LOEO+SHAP)放前面, 弱项(matbench/跨数据集)作为"边界讨论"。

## 最终诚实判断

**你的四个问题, 直答**:

1. **期刊看重什么?** 数据严谨>方法创新>物理洞察>性能。我们满足三区下限。

2. **要写原创算法章节吗?** **强烈建议**。这是把论文从"勉强三区"提升到"稳三区冲二区"的最大杠杆。当前PGML太简单, 升级为PCRL(带数学推导)是高价值投入。

3. **效果不好就不达标?** 不完全对。形成能够, 凸包能配合洞察够, matbench差但放局限讨论。

4. **代码百分百正确?** **不是。5个P0 bug必须修。** 尤其PGML物理贡献占比(58%)和新候选表数据是错的, 修复前不能写进论文。

**一句话总结: 项目骨架好, 但需要 (1)修5个bug (2)写原创算法章节 (3)精心组织叙事。这三步做完, 是一篇扎实的三区论文。**
