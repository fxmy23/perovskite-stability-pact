## P2 中等问题 (修复/辩护)

### M2 stacking元学习器Ridge过简单 — 辩护保留
**分析**: 基模型相关性0.99, Ridge(线性)足够。非线性元学习器(RF/GBDT)在
高相关基模型上收益有限, 且增加过拟合风险。
**决定**: 保留Ridge (stacking标准稳健选择), 论文注明"元学习器用Ridge,
  因基模型高相关(0.99)使非线性元学习器收益有限"。

### M3 物理层在解耦后语义变窄 — 已在P1-2诚实重定位

### M4 ★ 固定超参乐观偏差 +0.0053 (实测确认!)
**实测**: 固定Optuna超参 R²=0.9150 vs 折内嵌套CV R²=0.9097。
  差值 +0.0053 > 0.005阈值 → **有乐观偏差**。
**修复方案**:
  - 选项A (最严谨): 论文报嵌套CV R²=0.9097 (诚实但数值降低)
  - 选项B (折中): 报固定超参R²=0.9150, 但注明"超参在全量数据搜索,
    嵌套CV验证显示~0.005乐观偏差"
  - **推荐B**: 三区论文普遍用固定超参, 注明偏差即可。
  论文写: "Hyperparameters were selected via Optuna on the full dataset;
  nested CV estimates ~0.005 optimistic bias, within bootstrap CI."
**影响**: pact_final报的R²0.914有~0.005乐观, 真实泛化R²≈0.909-0.914。
  bootstrap CI [0.907,0.922]已覆盖此范围, 故CI报告是诚实的。
